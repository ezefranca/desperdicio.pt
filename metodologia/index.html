Visão Geral

O desperdicio.pt é um projecto de jornalismo de dados construído a partir do arquivo do Público preservado pelo Arquivo.pt. Utilizamos uma combinação de APIs, web scraping, e processamento de linguagem natural para construir um corpus de 337 artigos identificados entre 1999 e 2026, dos quais 127 artigos core ficaram no conjunto final usado no site depois da validação manual.

Fontes de Dados

1. Arquivo.pt Text Search API

A principal fonte de dados históricos é a API de pesquisa textual do Arquivo.pt, que permite pesquisar no conteúdo arquivado de websites portugueses.

GET https://arquivo.pt/textsearch?q={query}&siteSearch=www.publico.pt

Parâmetros utilizados:

Parâmetro Valor Descrição
q 16 variantes de query Termos de pesquisa em PT-PT, PT-BR, EN e variantes políticas
siteSearch www.publico.pt Restrição ao domínio do Público
from / to 1996-2025 Intervalo temporal (chunked por ano)
maxItems 100 Itens por página
offset 0-500 Paginação (máx 500 páginas por slice)

Queries de pesquisa:

  • "desperdício alimentar" (principal)
  • "desperdício de alimentos"
  • "perdas e desperdício alimentar"
  • "redução do desperdício alimentar"
  • "combate ao desperdício alimentar"
  • "sobras de comida"
  • "aproveitamento de sobras"
  • "desperdício de comida" (PT-BR)
  • "desperdício de alimentos no varejo" (PT-BR)
  • "perdas e desperdício de alimentos"
  • "food waste" (EN)
  • "food loss and waste" (EN)
  • "food loss" (EN)
  • "FLW"
  • "Banco Alimentar desperdício"
  • "redistribuição alimentar"

2. Público.pt Live API

Para artigos mais recentes (2020-2026), utilizamos a API interna do Público.pt que permite pesquisa no arquivo vivo do jornal.

GET https://www.publico.pt/api/list/search?query={query}

3. HTML Scraping

Para enriquecer os metadados, fizemos scraping das páginas de artigos usando BeautifulSoup para extrair: título completo, data de publicação, secção, autores, contagem de palavras, e detecção de paywall.

Classificação de Relevância

Cada artigo é classificado como core, maybe ou noise por um sistema heurístico aditivo em scripts/relevance_filter.py. A pontuação final combina padrões positivos e negativos aplicados sobre título e texto extraído.

Para manter esta página fiel ao repositório, o bloco seguinte resume a regra implementada em código em vez de apresentar uma lista manual incompleta de regex.

Resumo da Regra

# Para cada padrão positivo encontrado
score += 3

# Para cada padrão negativo encontrado
score -= 2

# Bónus explícito quando a frase exacta aparece
if "desperdício alimentar" in texto:
    score += 6

# Conversão final
score >= 9   → core
score >= 4   → maybe
score <  4   → noise

Famílias de padrões usadas

Entre os positivos entram variantes de desperdício, food waste, food loss, sobras, doação, banco alimentar, validade e deitar fora. Entre os negativos entram contextos de ruído recorrente, como pesca, futebol, bolsa e outros usos não alimentares do termo. 

Fonte canónica desta heurística:
scripts/relevance_filter.py

O corpus final usado no site não depende só desta etapa.
Depois da heurística, existe uma curadoria manual documentada
em corpus_curation.json e corpus_validation.json.

Validação Final do Corpus

A versão final usada na candidatura não ficou dependente apenas da heurística automática. Depois do export, aplicámos um passo explícito de curadoria final com scripts/curate_corpus.py.

Esse passo faz três coisas: normaliza datas para ISO, remove falsos positivos óbvios dentro de core e promove peças claramente focadas em desperdício alimentar que tinham ficado subavaliadas.

  • Lista de curadoria: assets/data/corpus_curation.json
  • Relatório técnico: assets/data/corpus_validation.json
  • Nota curta de validação: VALIDACAO-FINAL-CORPUS.md

Resultado final fechado: 337 artigos processados, 127 artigos core, 14 anos com cobertura publicada e intervalo editorial entre 2011 e 2026.

Como Executar

Requisitos

  • Python 3.10+
  • requests
  • beautifulsoup4
  • sqlite3 (built-in)
  • Node.js opcional para validação rápida de sintaxe JS

Instalação

# Clonar o repositório
git clone https://github.com/ezefranca/desperdicio.pt.git
cd desperdicio.pt

# Criar ambiente virtual
python3 -m venv .venv
source .venv/bin/activate  # Linux/Mac
.venv\Scripts\activate     # Windows

# Instalar dependências
pip install requests beautifulsoup4

Execução do Pipeline

# 1. Fetch histórico via Arquivo.pt (opcional, complementar)
python3 scripts/fetch_publico_arquivo_metadata.py

# 2. Fetch do arquivo vivo do Público
python3 scripts/fetch_publico_json_search.py

# 3. Enriquecer com metadados HTML
python3 scripts/enrich_publico_from_html.py

# 4. Classificar relevância
python3 scripts/relevance_filter.py

# 5. Exportar corpus para assets/data
python3 scripts/export_corpus.py

# 6. Aplicar curadoria final e regenerar timeline
python3 scripts/curate_corpus.py

# 7. Construir storyline e bundles
python3 scripts/build_storyline.py
python3 scripts/enrich_storyline.py
python3 scripts/build_bundled_storyline.py

# 8. Anexar URLs verificadas do Arquivo.pt e gerar corpus_core.json
python3 scripts/attach_arquivo_pt_urls.py

Notas: o pipeline histórico inclui sistema de checkpoint/resume. O snapshot storyline_bundled_full.json é uma versão editorial preferencial versionada no repositório; a cadeia automática reproduzível termina em storyline_bundled.json.

Executar o website localmente

python3 -m http.server 4173

Depois abrir http://127.0.0.1:4173/. O site tenta carregar primeiro storyline_bundled_full.json, depois storyline_bundled.json e por fim storyline.json.

Limitações

  • Cobertura pré-2010: O arquivo anterior a 2010 devolve sobretudo referências dispersas (PDFs, dossiers políticos, outros usos do termo). O desperdício alimentar ainda não tinha cobertura consistente como tema autónomo.
  • Paywall: Alguns artigos do Público.pt estão atrás de paywall, limitando o acesso ao texto completo.
  • Deduplicação: O Arquivo.pt pode conter múltiplas versões do mesmo artigo (snapshots diferentes). Aplicamos deduplicação por URL normalizado.
  • Classificação heurística: O filtro por regex melhora muito a precisão, mas não substitui revisão editorial. É por isso que o conjunto final depende de uma etapa explícita de curadoria documentada.
  • Indicadores externos de iniciativas: As métricas públicas de organizações civis mudam ao longo do tempo e não seguem o mesmo método de reporte. O site usa páginas oficiais e evita agregações enganosas.

Como usar este projecto

O desperdicio.pt foi desenhado para funcionar como editorial, dataset e caso de estudo ao mesmo tempo.

  • Professores: usar o hero, o Arquivo Profundo e o jogo para trabalhar sustentabilidade, literacia mediática e preservação digital.
  • Jornalistas: usar a metodologia como modelo replicável para outros temas acompanhados ao longo do tempo no Arquivo.pt.
  • Investigadores: reutilizar corpus_core.json, corpus.json, timeline.json e corpus_validation.json como base para análise longitudinal.
  • ONG e decisores locais: usar a timeline legislativa e os exemplos de iniciativas para discutir implementação e política pública.

Kit de Avaliação

Para apresentações curtas ao júri, o projecto já tem um percurso de demonstração com cinco passos e um conjunto estável de links de apoio.

  1. Abrir a página principal e confirmar o recorte: 127 artigos core, 20 diplomas e 14 anos de cobertura publicada.
  2. Ir à secção sobre dependência do Arquivo.pt e abrir uma versão preservada.
  3. Mostrar o Arquivo Profundo para evidenciar como, antes de 2011, o tema ainda surgia de forma dispersa.
  4. Abrir esta metodologia e a secção de validação final do corpus.
  5. Fechar no repositório e nos ficheiros públicos corpus_core.json e corpus_validation.json.

O resumo deste material está também em KIT-DE-AVALIACAO.md.
