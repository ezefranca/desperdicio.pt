Metodologia

Como construímos o desperdicio.pt: das APIs do Arquivo.pt ao editorial final. Uma documentação técnica completa do pipeline de dados, ferramentas utilizadas, e estrutura do corpus jornalístico.

Visão Geral

O desperdicio.pt é um projecto de jornalismo de dados construído a partir do arquivo do Público preservado pelo Arquivo.pt. Utilizamos uma combinação de APIs, web scraping, e processamento de linguagem natural para construir um corpus de 337 artigos sobre desperdício alimentar publicados entre 1999 e 2026.

Fontes de Dados

1. Arquivo.pt Text Search API

A principal fonte de dados históricos é a API de pesquisa textual do Arquivo.pt, que permite pesquisar no conteúdo arquivado de websites portugueses.

GET https://arquivo.pt/textsearch?q={query}&siteSearch=www.publico.pt

Parâmetros utilizados:

Parâmetro Valor Descrição
q 8 variantes de query Termos de pesquisa em PT-PT, PT-BR
siteSearch www.publico.pt Restrição ao domínio do Público
from / to 1996-2025 Intervalo temporal (chunked por ano)
maxItems 100 Itens por página
offset 0-500 Paginação (máx 500 páginas por slice)

Queries de pesquisa:

  • "desperdício alimentar" (principal)
  • "desperdício de alimentos"
  • "perdas e desperdício alimentar"
  • "combate ao desperdício alimentar"
  • "sobras de comida"
  • "desperdício de comida" (PT-BR)
  • "desperdício de alimentos no varejo" (PT-BR)
  • "food waste" (EN)

2. Público.pt Live API

Para artigos mais recentes (2020-2026), utilizamos a API interna do Público.pt que permite pesquisa no arquivo vivo do jornal.

GET https://www.publico.pt/api/list/search?query={query}

3. HTML Scraping

Para enriquecer os metadados, fizemos scraping das páginas de artigos usando BeautifulSoup para extrair: título completo, data de publicação, secção, autores, contagem de palavras, e detecção de paywall.

Classificação de Relevância

Cada artigo é classificado como core (relevante) ou noise (ruído) usando um sistema de padrões regex. Artigos com mais padrões positivos que negativos são marcados como "core".

Padrões Positivos

POSITIVE_PATTERNS = [
    r"desperdício\s+(alimentar|de\s+alimentos|de\s+comida)",
    r"food\s+waste",
    r"doação\s+de\s+(alimentos|géneros)",
    r"data\s+de\s+validade",
    r"fruta\s+feia",
    r"too\s+good\s+to\s+go",
    r"sobras\s+de\s+comida",
    r"redistribuição\s+alimentar"
]

Padrões Negativos

NEGATIVE_PATTERNS = [
    r"pesca",
    r"futebol",
    r"bolsa",
    r"política\s+externa",
    r"desperdício\s+de\s+energia",
    r"desperdício\s+de\s+água",
    r"desperdício\s+de\s+dinheiro"
]

Como Executar

Requisitos

  • Python 3.10+
  • requests
  • beautifulsoup4
  • sqlite3 (built-in)

Instalação

# Clonar o repositório
git clone https://github.com/ezefranca/desperdicio.pt.git
cd desperdicio.pt

# Criar ambiente virtual
python -m venv venv
source venv/bin/activate  # Linux/Mac
venv\Scripts\activate     # Windows

# Instalar dependências
pip install requests beautifulsoup4

Execução do Pipeline

# 1. Fetch dados do Arquivo.pt (pode demorar ~30min)
python scripts/fetch_publico_arquivo_metadata.py

# 2. Enriquecer com metadados HTML
python scripts/enrich_publico_from_html.py

# 3. Classificar relevância
python scripts/relevance_filter.py

# 4. Exportar corpus
python scripts/export_corpus.py

# 5. Construir storyline e bundles
python scripts/build_storyline.py
python scripts/build_bundled_storyline.py

Nota: O pipeline inclui sistema de checkpoint/resume. Se a execução for interrompida, pode ser retomada do ponto onde parou.

Limitações

  • Cobertura pré-2010: O arquivo anterior a 2010 é maioritariamente ruído (PDFs, dossiers políticos). O desperdício alimentar ainda não era tema mediatizado.
  • Paywall: Alguns artigos do Público.pt estão atrás de paywall, limitando o acesso ao texto completo.
  • Deduplicação: O Arquivo.pt pode conter múltiplas versões do mesmo artigo (snapshots diferentes). Aplicamos deduplicação por URL normalizado.
  • Discrepâncias numéricas: Algumas fontes reportam valores ligeiramente diferentes (184kg vs 185kg per capita). O editorial usa valores do Eurostat.
